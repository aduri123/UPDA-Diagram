Overview

This flow helps teams stand up a working agentic GIS capability quickly: define a narrow use case, build a reasoning agent, connect ArcGIS tools and data, and ship with CI/CD and governance.

1) Define Use Case & KPI

  • Pick one outcome (e.g., zoning compliance for a parcel).
  • Write acceptance criteria & KPI (precision/latency/coverage).
  • Identify inputs: parcels, titles, policies, imagery, IoT.

2) Build a Reasoning Agent

  • LLM + RAG over policy docs; citations required.
  • Knowledge Graph (openCypher) for relationships & rules.
  • Tool-calling to run spatial queries & validations.
MATCH (p:Parcel)-[:LOCATED_IN]->(z:Zone)
WHERE p.id = $parcelId AND z.setback_m <= 3
RETURN p.id AS parcelId, z.name AS zone, z.setback_m AS rule;

3) Connect ArcGIS Tools (Tool Agents)

  • ArcGIS Feature/Map/Image services via REST & Python API.
  • ArcGIS Pro GP tools exposed through FastAPI endpoints.
  • Feature store by entity_id (parcel_id, asset_id, case_id).

4) Orchestrate with LangGraph / MCP

  • Graph of agents: Retrieval → Reasoning → Validation → Writer.
  • Human-in-the-loop gates for risk-based approvals.
  • ABAC policies on tools & data; OAuth2/OIDC at gateway.

5) Deliverables & UX

  • Dashboards (ArcGIS Dashboards / Experience Builder).
  • Explain packets: citations, rule paths, KG evidence.
  • Notifications & work orders via webhooks (EAM/CRM).

6) CI/CD, Testing, & Ops

  • IaC (Terraform / ArcGIS Admin API) from staging → prod.
  • TestGen: synthetic data + golden answers in CI.
  • Backups, tag-based restores, cost & model routing.
