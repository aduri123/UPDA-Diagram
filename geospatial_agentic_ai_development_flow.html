Geospatial Agentic AI System Development Flow

Define → Build → Integrate → Orchestrate → Deliver → Operate.

1) Define Use Case & KPI

  • Pick one outcome (e.g., zoning compliance for a parcel).
  • Acceptance criteria & KPI (precision, latency, coverage).
  • Inputs: parcels, titles, zoning, imagery/LiDAR, IoT, policies.

2) Build a Reasoning Agent

  • LLM + RAG (policy docs with citations).
  • Knowledge Graph (openCypher) for rule paths & relationships.
  • Tool-calling for spatial validation & queries.
MATCH (p:Parcel)-[:LOCATED_IN]->(z:Zone)
WHERE p.id = $parcelId AND z.setback_m <= 3
RETURN p.id AS parcelId, z.name AS zone, z.setback_m AS rule;

3) Connect ArcGIS Tools (Tool Agents)

  • ArcGIS REST & Python API (Feature/Map/Image services).
  • ArcGIS Pro GPTools via FastAPI endpoints.
  • Feature store by entity_id (parcel_id, asset_id, case_id).

4) Orchestrate with LangGraph / MCP

  • Graph: Retrieval → Reasoning → Validation → Writer.
  • Human-in-the-loop gates (risk-based approvals).
  • ABAC on tools & data; OAuth2/OIDC at gateway.

5) Deliverables & UX

  • Dashboards (ArcGIS Dashboards / Experience).
  • Explain packets: citations, rule paths, KG evidence.
  • Notifications & work orders via webhooks (EAM/CRM).

6) CI/CD, Testing, & Ops

  • IaC (Terraform / ArcGIS Admin API): staging → prod.
  • TestGen with synthetic data & golden answers in CI.
  • Backups/restores with tags; cost & model routing.
