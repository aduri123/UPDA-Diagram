Orchestration (LangGraph + LangChain)

Rule of thumb: LangGraph orchestrates the DAG & control flow (branches, approvals, retries). Inside nodes, use LangChain for RAG steps, tool wrappers, and retrievers.

LangGraph
  • Branch & merge parallel tasks (e.g., zoning + impact analysis)
  • Human-in-the-loop approval gates for high-risk actions
  • Route intents to agents via typed nodes
LangChain
  • Wrap ArcGIS/SDI/BIM services as tools (WMS/WFS/WCS, Feature Services, IFC parsers)
  • Compose RAG chains (retrieve ‚Üí ground ‚Üí generate ‚Üí verify) with citations
  • Build tool-calling agents executing ArcPy/GP tasks & SQL/OpenCypher queries

Concrete Example ‚Äî Permit Validation (City/Municipal)

  • LG‚Äë1: Router ‚Äî classify intent=validate_permit; pass parcel_id
  • LC‚Äë2: SQL Pre‚ÄëFilter ‚Äî Query Layer: zoning='RES' AND area>=500
  • LC‚Äë3: Policy RAG ‚Äî retrieve zoning clauses; rule JSON + citations (EN/AR)
  • LG‚Äë4: Rule Check ‚Äî deterministic checks (setbacks/FAR/coverage)
  • LC‚Äë5: KG Traverse ‚Äî openCypher: (Parcel)-[:HAS_PERMIT]->(Permit) AND NOT (Permit)-[:HAS_INSPECTION]->(:Inspection)
  • LG‚Äë6: Plan & Diff ‚Äî build change‚Äëset + evidence report
  • LG‚Äë7: Approval Gate ‚Äî human-in-the-loop for high‚Äërisk parcels
  • LC‚Äë8/9: Apply & Explain ‚Äî guarded write + KPIs & citations
Legend: LG = LangGraph node (DAG/control); LC = LangChain step (RAG/tools).

Query Layers & Knowledge Graph

-- SQL (Query Layers)
SELECT parcel_id, area_m2, zoning, title_status
FROM cadastral_parcels
WHERE municipality = 'City/MunicipalArea'
  AND title_status = 'Active'
  AND area_m2 >= 500;

// openCypher (Knowledge Graph)
MATCH (o:Owner)-[r:OWNS]->(p:Parcel)
WHERE r.share >= 0.5
  AND NOT (p)-[:HAS_LIENS]->(:Lien {status:'Active'})
RETURN p.parcelId AS parcelId, o.ownerId AS ownerId, r.share AS share;
