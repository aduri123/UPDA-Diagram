Orchestration (LangGraph + LangChain)

Control
Rule of thumb: Use LangGraph to orchestrate the DAG & control flow (branches, approvals, retries). Inside nodes, use LangChain for RAG steps, tool wrappers, retrievers. LangChain can also orchestrate simple chains; RAG can be built without it‚Äîthe split keeps things clear.

LangGraph

  • üîÄ Branch & merge parallel tasks (e.g., zoning check + impact analysis) then reconcile.
  • üßç Human-in-the-loop approval gates for high-risk actions (permits/overrides).
  • üß≠ Route intents to agents (Planning, Vision, Code, Conversation) via typed nodes.

LangChain

  • üß∞ Wrap ArcGIS/SDI/BIM services as tools (WMS/WFS/WCS, Feature Services, IFC parsers).
  • üìö Compose RAG chains (retrieve ‚Üí ground ‚Üí generate ‚Üí verify) for policy answers.
  • üõ†Ô∏è Build tool-calling agents that execute ArcPy/GP tasks and SQL/OpenCypher queries.

Government Orchestration Examples

DAGs
  • Permit Validation DAG (with RAG) ‚Äî Intent ‚Üí SQL pre-filter (eligible parcels/applicants) ‚Üí RAG grounding (retrieve applicable regulations + citations) ‚Üí Rule Engine checks ‚Üí KG traverse (missing inspections/conflicts) ‚Üí Plan (change-set) ‚Üí Approval gate ‚Üí Apply edits ‚Üí Evidence report.
  • Land Registry ‚Äî Title Transfer Validation ‚Äî Intake (new transfer) ‚Üí RAG (title rules, encumbrance clauses) ‚Üí KG ownership & lien traversal ‚Üí SQL parcel/lot checks (area, zoning) ‚Üí Risk score ‚Üí Notary approval gate ‚Üí Update titles & audit log.
  • Real Estate ‚Äî AVM & Listing Compliance ‚Äî SQL comparable sales & features ‚Üí Code Agent ML inference (AVM) ‚Üí RAG disclosure/advertising rules ‚Üí Listing check (violations) ‚Üí Publish price band + compliance report.
  • Asset Maintenance Scheduling ‚Äî Event (sensor alert or complaint) ‚Üí SQL filter assets by class/condition ‚Üí KG fetch work history & dependencies ‚Üí Route to crew (capacity, location) ‚Üí Generate work order ‚Üí Notify citizen.
  • Health Inspection Prioritization ‚Äî Intake (restaurant permits) ‚Üí RAG policy fetch (risk scoring) ‚Üí SQL candidate facilities by violations ‚Üí KG join complaints & past penalties ‚Üí Schedule inspections ‚Üí Publish dashboard.
  • Urban Studies ‚Äî Accessibility & Equity ‚Äî Intent ‚Üí RAG policy goals ‚Üí Network isochrones (5/10/15 min walk) ‚Üí Join demographics ‚Üí KPIs (% within 15 min, access Gini, land‚Äëuse mix entropy) ‚Üí Scenario diff ‚Üí Dashboard & report.

üìå Concrete Example ‚Äî Permit Validation (City/Municipal)

LG/LC
  1. LG‚Äë1: Router (LangGraph) ‚Äî classify intent=validate_permit; pass parcel_id.
  2. LC‚Äë2: SQL Pre‚ÄëFilter (LangChain tool) ‚Äî Query Layer: candidate parcels (zoning='RES' AND area>=500).
  3. LC‚Äë3: Policy RAG (LangChain) ‚Äî retrieve zoning clauses; output rule JSON + citations (Arabic/English).
  4. LG‚Äë4: Rule Check Node (LangGraph) ‚Äî deterministic checks using rule JSON (setbacks/FAR/coverage).
  5. LC‚Äë5: KG Traverse (LangChain tool) ‚Äî openCypher: (Parcel)‚Äë[:HAS_PERMIT]‚Üí(Permit) AND NOT (Permit)‚Äë[:HAS_INSPECTION]‚Üí(:Inspection).
  6. LG‚Äë6: Plan & Diff (LangGraph) ‚Äî build change‚Äëset (flags, annotations) + generate evidence report.
  7. LG‚Äë7: Approval Gate (LangGraph) ‚Äî human‚Äëin‚Äëthe‚Äëloop for high‚Äërisk parcels.
  8. LC‚Äë8: Apply (LangChain tool) ‚Äî guarded write to Feature Service (MCP scope enforced).
  9. LC‚Äë9: Explain (LangChain) ‚Äî return KPIs, citations, rule path, and KG proof.
Legend: LG = LangGraph node (DAG/control); LC = LangChain step (RAG/tools).

Query Layers & Knowledge Graph

-- SQL (Query Layers)
SELECT parcel_id, area_m2, zoning, title_status
FROM cadastral_parcels
WHERE municipality = 'City/MunicipalArea'
  AND title_status = 'Active'
  AND area_m2 >= 500;

// openCypher (Knowledge Graph)
MATCH (o:Owner)-[r:OWNS]->(p:Parcel)
WHERE r.share >= 0.5
  AND NOT (p)-[:HAS_LIENS]->(:Lien {status:'Active'})
RETURN p.parcelId AS parcelId, o.ownerId AS ownerId, r.share AS share;
