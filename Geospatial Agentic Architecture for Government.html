Orchestration (LangGraph + LangChain)

LangGraph

  • Branch & merge parallel tasks (zoning check + impact analysis) then reconcile.
  • Human-in-the-loop approval gates for high-risk actions (permits/overrides).
  • Route intents to agents (Planning, Vision, Code, Conversation) via typed nodes.
Government orchestration examples
  1. Permit Validation DAG (with RAG): Intent → SQL pre-filter (eligible parcels/applicants) → RAG grounding (retrieve applicable regulations + citations) → Rule Engine checks → KG traverse (missing inspections/conflicts) → Plan (change-set) → Approval gate → Apply edits → Evidence report.
  2. Land Registry — Title Transfer Validation: Intake (new transfer) → RAG (title rules, encumbrance clauses) → KG ownership & lien traversal → SQL parcel/lot checks (area, zoning) → Risk score → Notary approval gate → Update titles & audit log.
  3. Real Estate — AVM & Listing Compliance: SQL comparable sales & features → Code Agent ML inference (AVM) → RAG disclosure/advertising rules → Listing check (violations) → Publish price band + compliance report.
  4. Asset Maintenance Scheduling: Event (sensor alert or complaint) → SQL filter assets by class/condition → KG fetch work history & dependencies → Route to crew (capacity, location) → Generate work order → Notify citizen.
  5. Health Inspection Prioritization: Intake (restaurant permits) → RAG policy fetch (risk scoring) → SQL candidate facilities by violations → KG join complaints & past penalties → Schedule inspections → Publish dashboard.
  6. Urban Studies — Accessibility & Equity Assessment: Intent → RAG policy goals (e.g., 15‑minute access standards) → Network isochrones (5/10/15 min walk) for clinics, schools, parks → Join demographics (census/HH surveys) → KPIs (% population within 15 min, access Gini, land‑use mix entropy) → Scenario diff (new clinic/line) → Dashboard & report.

LangChain

  • Wrap ArcGIS/SDI/BIM services as tools (WMS/WFS/WCS, Feature Services, IFC parsers).
  • Compose RAG chains (retrieve → ground → generate → verify) for policy answers.
  • Build tool-calling agents that execute ArcPy/GP tasks and SQL/OpenCypher queries.
Agent & tool examples (government)
  1. Policy Q&A RAG: Retrieve bilingual zoning clauses (+ citations) → Ground a parcel query → Generate answer → Verify against rule JSON (setbacks/FAR).
  2. SQL/KG router agent: If task is set-based (e.g., "find RES parcels > 500 m²") → run SQL Query Layer; if multi-hop ("permits without inspections") → run openCypher on KG.
  3. GP/ArcPy tool-caller: Buffer schools, overlay noise zones, compute OSR; write outputs as Feature Service layers behind approval.
  4. Reviewer tool: Run Data Reviewer checks via tool binding and return issue layers to the Audit agent.
Routing rule: set‑based → SQL; relationship/multi‑hop → KG.
Rule of thumb: Use LangGraph to orchestrate the DAG & control flow (branches, approvals, retries). Inside those nodes, use LangChain to build the RAG steps, tool wrappers, and retrievers.
Note: LangChain can also orchestrate simple chains and RAG can be built without LangChain; this split keeps the architecture clear.

📌 Concrete example — Permit Validation (City/Municipal)

  1. LG‑1: Router (LangGraph) — classify intent=validate_permit; pass parcel_id.
  2. LC‑2: SQL Pre‑Filter (LangChain tool) — Query Layer: candidate parcels (zoning='RES' AND area>=500).
  3. LC‑3: Policy RAG (LangChain) — retrieve zoning clauses; output rule JSON + citations (Arabic/English).
  4. LG‑4: Rule Check Node (LangGraph) — deterministic checks using rule JSON (setbacks/FAR/coverage).
  5. LC‑5: KG Traverse (LangChain tool) — openCypher: (Parcel)‑[:HAS_PERMIT]→(Permit) AND NOT (Permit)‑[:HAS_INSPECTION]→(:Inspection).
  6. LG‑6: Plan & Diff (LangGraph) — build change‑set (flags, annotations) + generate evidence report.
  7. LG‑7: Approval Gate (LangGraph) — human‑in‑the‑loop for high‑risk parcels.
  8. LC‑8: Apply (LangChain tool) — guarded write to Feature Service (MCP scope enforced).
  9. LC‑9: Explain (LangChain) — return KPIs, citations, rule path, and KG proof.
Legend: LG = LangGraph node (DAG/control); LC = LangChain step (RAG/tools).

Query Layers & Knowledge Graph

SQL (Query Layers)
SELECT parcel_id, area_m2, zoning, title_status
FROM cadastral_parcels
WHERE municipality = 'City/MunicipalArea' AND title_status = 'Active' AND area_m2 >= 500;
openCypher (Knowledge Graph)
MATCH (o:Owner)-[r:OWNS]->(p:Parcel)
WHERE r.share >= 0.5 AND NOT (p)-[:HAS_LIENS]->(:Lien {status:'Active'})
RETURN p.parcelId AS parcelId, o.ownerId AS ownerId, r.share AS share;
