Orchestration (LangGraph + LangChain) — Control

Rule of thumb: Use LangGraph to orchestrate the DAG & control flow (branches, approvals, retries). Inside nodes, use LangChain for RAG steps, tool wrappers, retrievers. LangChain can also orchestrate simple chains; RAG can be built without it—the split keeps things clear.

LangGraph

  • Branch & merge parallel tasks (e.g., zoning check + impact analysis) then reconcile.
  • Human-in-the-loop approval gates for high-risk actions (permits/overrides).
  • Route intents to agents (Planning, Vision, Code, Conversation) via typed nodes.

LangChain

  • Wrap ArcGIS/SDI/BIM services as tools (WMS/WFS/WCS, Feature Services, IFC parsers).
  • Compose RAG chains (retrieve → ground → generate → verify) for policy answers.
  • Build tool-calling agents that execute ArcPy/GP tasks and SQL/OpenCypher queries.
Government Orchestration Examples (DAGs)
  1. Permit Validation DAG (with RAG) — Intent → SQL pre-filter (eligible parcels/applicants) → RAG grounding (retrieve applicable regulations + citations) → Rule Engine checks → KG traverse (missing inspections/conflicts) → Plan (change-set) → Approval gate → Apply edits → Evidence report.
  2. Land Registry — Title Transfer Validation — Intake (new transfer) → RAG (title rules, encumbrance clauses) → KG ownership & lien traversal → SQL parcel/lot checks (area, zoning) → Risk score → Notary approval gate → Update titles & audit log.
  3. Real Estate — AVM & Listing Compliance — SQL comparable sales & features → Code Agent ML inference (AVM) → RAG disclosure/advertising rules → Listing check (violations) → Publish price band + compliance report.
  4. Asset Maintenance Scheduling — Event (sensor alert or complaint) → SQL filter assets by class/condition → KG fetch work history & dependencies → Route to crew (capacity, location) → Generate work order → Notify citizen.
  5. Health Inspection Prioritization — Intake (restaurant permits) → RAG policy fetch (risk scoring) → SQL candidate facilities by violations → KG join complaints & past penalties → Schedule inspections → Publish dashboard.
  6. Urban Studies — Accessibility & Equity — Intent → RAG policy goals → Network isochrones (5/10/15 min walk) → Join demographics → KPIs (% within 15 min, access Gini, land-use mix entropy) → Scenario diff → Dashboard & report.

Query Layers & Knowledge Graph

SQL (Query Layers)
SELECT parcel_id, area_m2, zoning, title_status
FROM cadastral_parcels
WHERE municipality = 'City/MunicipalArea'
  AND title_status = 'Active'
  AND area_m2 >= 500;
openCypher (Knowledge Graph)
MATCH (o:Owner)-[r:OWNS]->(p:Parcel)
WHERE r.share >= 0.5
  AND NOT (p)-[:HAS_LIENS]->(:Lien {status:'Active'})
RETURN p.parcelId AS parcelId, o.ownerId AS ownerId, r.share AS share;
